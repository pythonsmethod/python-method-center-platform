# Client_Cabinet_Architecture_v1

## Статус документа

Документ №3 Тома 2 «Архитектура запуска».

Основан на:

- Launch Architecture Foundation v1
- Support System Architecture v1

-----

# 1. Назначение кабинета

Личный кабинет является основным пространством взаимодействия клиента с центром.

Кабинет представляет собой не профиль пользователя, а дом его кейса.

Все взаимодействие клиента с центром происходит через личный кабинет.

-----

# 2. Главная панель

Главная панель является стартовым экраном после входа в систему.

Основная задача:

Ответить клиенту на вопрос:

«Что сейчас происходит с моим кейсом?»

-----

## Блок: Статус кейса

Отображает:

- текущий статус кейса;
- активность сопровождения;
- тип программы;
- дату начала;
- дату окончания периода сопровождения.

Примеры:

- Сопровождение активно
- Анализ завершён
- Сопровождение не активно

-----

## Блок: Следующий шаг

Показывает одно главное действие.

Примеры:

- Завершить онбординг
- Загрузить анализы
- Ответить на запрос Карена
- Ожидается ответ Карена
- Продлить сопровождение

-----

## Блок: Последняя активность

Отображает последние действия по кейсу.

Примеры:

- Карен ответил на обращение
- Загружены анализы
- Создано обращение
- Активировано сопровождение

-----

## Блок: Быстрые действия

Основные кнопки:

- Создать обращение
- Загрузить анализы
- Открыть рекомендации
- Спросить AI

-----

## Блок: Маршрут клиента

Отображает текущее направление:

- Восстановление
- Реабилитация
- Сохранение

-----

# 3. Раздел «Мой кейс»

Содержит:

- профиль клиента;
- историю ситуации;
- цели клиента;
- статус кейса;
- периоды сопровождения.

-----

# 4. Раздел «Обращения к Карену»

Доступен только при активном сопровождении.

Позволяет:

- создавать обращения;
- прикреплять файлы;
- отслеживать статусы;
- получать ответы Карена.

Используется система обращений, а не обычный чат.

-----

# 5. Раздел «Документы и анализы»

Хранит:

- анализы;
- выписки;
- медицинские документы;
- дополнительные материалы.

Все документы сохраняются в истории кейса.

-----

# 6. Раздел «Рекомендации Карена»

Содержит:

- первичное заключение;
- рекомендации;
- обновления;
- корректировки.

Все рекомендации доступны после завершения сопровождения.

-----

# 7. Раздел «История сопровождения»

Отображает:

- периоды сопровождения;
- даты начала;
- даты окончания;
- историю активаций.

История сохраняется бессрочно.

-----

# 8. AI-Помощник

Доступен круглосуточно.

Помогает:

- ориентироваться в системе;
- находить документы;
- понимать следующий шаг;
- готовить обращения;
- объяснять статусы.

AI не заменяет решения Карена.

-----

# 9. Раздел «Оплаты»

Отображает:

- активный продукт;
- историю оплат;
- доступные продления;
- возможность приобрести сопровождение;
- возможность возобновить сопровождение.

-----

# 10. Раздел «Уведомления»

Содержит уведомления:

- новый ответ Карена;
- новое сообщение системы;
- изменение статуса обращения;
- завершение сопровождения;
- необходимость действия клиента.

-----

# 11. Принцип простоты

Главная панель не должна содержать:

- сложные графики;
- перегруженную статистику;
- избыточные таблицы;
- лишние показатели.

Кабинет должен помогать клиенту понимать:

- что происходит;
- что делать дальше;
- где находится его кейс.

-----

# 12. Статус Version 1

Документ определяет минимально необходимую архитектуру кабинета для запуска центра.

Дополнительные функции переносятся в Version 2 после запуска.
