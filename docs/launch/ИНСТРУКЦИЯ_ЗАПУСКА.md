# Инструкция запуска — что сделать руками (≈1–2 часа)

Код готов и лежит в ветке `claude/pythons-method-launch-audit-n9t4dh`. Чтобы принять первого клиента, осталось развернуть платформу. Делай по порядку.

## Шаг 1. Supabase (~15 мин)

Проект Supabase УЖЕ создан и настраивался ранее (июнь, этапы P0-005…P0-008):
схема, бакет `client-documents` и storage-политики уже применялись к живой базе.
Осталось довести до текущего состояния кода:

1. SQL Editor → выполни НОВУЮ миграцию `supabase/migrations/20260709120000_launch_hardening.sql`
   (добавлена 2026-07-09, в живой базе её точно нет).
2. Проверь, что применены две последние июньские миграции — выполни в SQL Editor:
   `select column_name from information_schema.columns where table_name = 'uploaded_documents' and column_name = 'document_status';`
   - если пусто → выполни `20260623030000_document_intake_queue_foundation.sql`,
     `20260623033000_document_status_insert_hardening.sql` и
     `20260623040000_staff_access_profile_role_hardening.sql`.
3. Проверь бакет: Storage → `client-documents` существует, приватный, политики
   SELECT/INSERT только для владельца пути (если нет — `supabase/storage_manual_setup.md`).
4. Скопируй из Settings → API: `Project URL`, `anon key`, `service_role key`
   (service_role — только в env Vercel, никогда в браузер и чаты).

## Шаг 2. Vercel (~20 мин)

1. vercel.com → New Project → импортируй репозиторий `pythonsmethod/python-method-center-platform`, ветку можно сразу `main` после мержа (или деплой из этой ветки для проверки).
2. Framework: Next.js (определится сам), корень — корень репозитория.
3. Environment Variables:
   - `NEXT_PUBLIC_SUPABASE_URL` = Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY` = anon key
   - `SUPABASE_SERVICE_ROLE_KEY` = service_role key
   - `NEXT_PUBLIC_STRIPE_PAYMENT_LINK_5W` / `_15W` — после шага 4 (можно добавить позже)
4. Deploy. Получишь URL вида `https://<проект>.vercel.app`.
5. Supabase → Authentication → URL Configuration:
   - Site URL: адрес деплоя
   - Redirect URLs: `https://<адрес>/auth/callback`

## Шаг 3. Профиль Карен (~5 мин)

1. Открой `https://<адрес>/login` и создай аккаунт для Карен (email + пароль).
2. Supabase → Table Editor → `profiles` → найди строку этого пользователя → поставь `role` = `admin`, `status` = `active`.
   (Если строки нет — сначала зайди под этим аккаунтом на `/onboarding` и отправь тестовую анкету, либо создай строку вручную с `id` пользователя из Authentication → Users.)
3. Проверь: под аккаунтом Карен открывается `/admin`.

## Шаг 4. Stripe (~15 мин)

1. В Stripe создай два Payment Link: «Сопровождение — 5 недель» и «Сопровождение — 15 недель» с ценами.
2. Вставь ссылки в Vercel env: `NEXT_PUBLIC_STRIPE_PAYMENT_LINK_5W`, `NEXT_PUBLIC_STRIPE_PAYMENT_LINK_15W` → Redeploy.
3. Когда клиент оплатит — на странице кейса в админке нажми «Записать оплату» (сумма, валюта, Stripe ID платежа).

## Шаг 5. Смоук-тест (~15 мин)

Пройди путь клиента с тестовым аккаунтом:

- [ ] `/` открывается, ссылки работают
- [ ] Регистрация и вход на `/login`
- [ ] `/legal/offer` показывает PDF оферты
- [ ] Анкета на `/onboarding`: оба чекбокса обязательны, после отправки — кабинет со статусом «Передан на изучение»
- [ ] Загрузка PDF в кабинете и кнопка «Открыть»
- [ ] Сообщение через «Написать команде»
- [ ] Под Карен: `/admin/cases` — кейс виден, анкета читается, статус меняется
- [ ] `/admin/documents` — документ открывается по ссылке
- [ ] `/admin/requests` — обращение видно, статус меняется
- [ ] Кнопки оплаты на `/payment` ведут на Stripe
- [ ] Под клиентом НЕ открывается `/admin` (страница «не найдено»)

Полный технический чек-лист — в `docs/deployment.md`.

## Шаг 6. Домен (можно после первого клиента)

Vercel → Domains → добавь `pythonmethodcenter.com` и `www`, в Namecheap поставь записи, которые покажет Vercel, затем добавь в Supabase Redirect URLs `https://pythonmethodcenter.com/auth/callback` и смени Site URL.

После шагов 1–5 можно отправлять первому клиенту ссылку на платформу. Ответы на обращения клиентов пока отправляются по email/телефону из карточки обращения — внутренняя переписка появится следующим этапом.
